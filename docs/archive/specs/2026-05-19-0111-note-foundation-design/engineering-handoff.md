## 一、功能识别

| 维度             | 内容                                                                                                                                                                                                                                                                                                                                |
| ---------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 用户问题         | 用户在已打开的本地记忆空间内，围绕一个 Memory 主题想用 Notion-like 沉浸式体验快速写一段 Markdown 笔记，并以普通文件保存到本地记忆空间文件夹（用户、agent、Finder 都能直接读取与修改），与现有录音 Segment 共享同一对象图与命名 / 删除 / 恢复合同                                                                                    |
| 功能位置         | 不开新 page；位于现有 Loaded Workspace shell：(1) 底部 ExpressionDock FAB Speed Dial 的"笔记"action；(2) Memory Studio selected Segment 的 `+` 菜单"笔记补充"action；(3) Memory Studio 内容区 MarkdownContentSurface 的 expand icon                                                                                                 |
| 用户入口         | FAB "笔记"启用（当前 disabled）；selected Segment `+` 菜单；MarkdownContentSurface expand icon                                                                                                                                                                                                                                      |
| 用户任务         | 创建 note Segment / Supplement、编辑笔记内容（含粘贴本地图片）、保存、重命名、删除/恢复                                                                                                                                                                                                                                             |
| 前置条件         | 已 open 记忆空间（workspaceHandle 有效）；已选 Memory（Memory Studio 状态，selected Segment 操作下要 +1 个 selected Segment）；未在录音；未在沉浸式录音/笔记编辑                                                                                                                                                                    |
| 真实 UI 元素来源 | 现有 ExpressionDock / MemoryStudio / SegmentActionsMenu / SegmentSupplementActionsMenu / RecordingOverlay 模板 + 新增 NoteEditorOverlay / MarkdownContentSurface / Segment card note preview                                                                                                                                        |
| 设计图来源       | 没有 PRD 阶段的标注设计图输入；当前结论来自用户文字（多模态拆解、约束、UI 方向）+ ADR 0006 + 设计 spec 22 节 + 录音端现有代码                                                                                                                                                                                                       |
| 状态影响 UI      | Segment kind 决定 player slot（audio = waveform / note = 占位文案）+ card preview（waveform glyph / line icon + 首段）+ More 菜单转录组；NoteEditorOverlay 内 subset detection 决定 normal vs raw mode                                                                                                                              |
| 数据读写         | 读 `segment.md` / `supplement.md` body；写 `.reo/drafts/segments/<id>/segment.md` 草稿；finalize 时 atomic move 到 `memories/<mem>/segments/<seg>/`；写 manifest（kind + bodyByteLength）；写 `attachments/<filename>`；外部冲突走 baselineContentHash 比对                                                                         |
| 权限影响         | 当前阶段无 auth；仅校验 workspaceHandle ownership + workspaceId 一致性 + lock usability；未来 Better Auth 接入留接入点                                                                                                                                                                                                              |
| 异常场景         | IPC 失败 / lock lost / 外部冲突 (`ERR_SEGMENT_CONTENT_STALE`) / attachment unsupported mime / too large / write failed / `reo-attachment://` 404 / 老 manifest 缺 kind 字段 / 应用崩溃后 recovery                                                                                                                                   |
| 工程约束         | Reo 当前 hard-code `kind: audio`；BlockNote markdown 转换 lossy 与 "Markdown 真源"硬约束冲突；`@blocknote/shadcn` 与 Reo Radix Portal 冲突；引入 `reo-attachment://` 是 Reo 第二个自定义 protocol；不引入 file watcher、SQLite、Sentry、Better Auth                                                                                 |
| 缺失项（待确认） | (1) sub-spec (b) 前必须收口 editor decision：BlockNote 0.51.1 与 Milkdown 7.21.1 均未通过当前 markdown-truth gate；(2) Memory Studio Segment strip 混排策略（当前默认 updatedAt 倒序混合）；(3) NoteEditorOverlay open 时是否允许折叠 sidebar/MemoryRail；(4) attachments stable filename sha256 前缀长度 → sub-spec (c) 代码前锁定 |

| 功能类型                                                                      | 是否涉及   | 判断依据                                                                                                                  |
| ----------------------------------------------------------------------------- | ---------- | ------------------------------------------------------------------------------------------------------------------------- |

| 编辑功能                                                                      | 主要类型   | Notion-like Markdown 编辑（BlockNote normal mode + Raw mode fallback）                                                    |
| 本地数据管理功能                                                              | 主要类型   | `segment.md` / `supplement.md` / `attachments/` 全部本地                                                                  |
| 桌面端系统能力功能                                                            | 涉及       | `reo-attachment://` custom protocol、IPC、主进程 lock                                                                     |
| 异步任务功能                                                                  | 涉及       | draft auto-save / finalize / write / discard / staleness check                                                            |
| 跨页面流程功能                                                                | 涉及       | FAB → NoteEditorOverlay 覆盖主区 → 关闭回到 Memory Studio（overlay layer，不开新窗口）                                    |
| 危险操作功能                                                                  | 涉及       | note Segment delete（grace period undo）/ Supplement delete（real mutation + restore toast）/ 冲突"保留我的修改"强制覆盖  |
| 搜索 / 筛选 / 发布 / 审批 / 支付 / 通知 / 数据看板 / AI / 多人协作 / 后台配置 | 不涉及     | 本 initiative 不在范围                                                                                                    |

| 维度     | 结论                                                                                                                                            |

| 核心组件 | NoteEditorOverlay / MarkdownContentSurface / Segment card note preview / Supplement tab note variant / 笔记 More 菜单 / 冲突 AlertDialog        |

| 状态切换 | 见 §九 状态机                                                                                                                                   |
| 数据输入 | 笔记 title（用户输入）+ body markdown（BlockNote / raw textarea）+ 粘贴图片 Blob                                                                |

| 权限规则 | 当前阶段无 auth；仅 workspaceHandle / workspaceId / lock 校验；未来 Better Auth 占位                                                            |
| 异常规则 | 见 §十二 边界情况补齐                                                                                                                           |
| 成功结果 | 笔记 segment.md 在 memories/ 下；Memory Studio 自动选中或切换 supplement tab；reopen 后可见                                                     |
| 失败结果 | draft + recovery marker 保留；toast 提示具体错误；用户可重试或丢弃                                                                              |
| 验收标准 | 见 §十三 验收标准                                                                                                                               |

## 七、产品功能说明

**1. 功能要解决的问题**
Reo 当前是 audio-only 的本地优先记忆空间。用户在围绕一个主题积累跨模态材料时缺一类核心模态——文字笔记。本功能让用户在已打开的记忆空间内，围绕一个 Memory 主题，以 Notion-like 沉浸式体验创建 / 编辑一段 Markdown 笔记，body 长期保存在本地 segment.md 中，外部 agent / Finder 可直接读写，与现有录音 Segment 共享同一对象图与命名 / 删除 / 恢复合同。

**2. 目标用户**
prosumer 创作者与思考者：Obsidian 用户、专业写作者、研究者、podcasters / video essayists、深度学习者、parent journalers、有强烈记录欲的领域专家。用户被允许、被预期：理解记忆空间是本地文件夹；接受 prompt-bridge 外部 agent 协作；不为零摩擦上手优化。

**3. 使用场景**
（a）在一本书 / 一个学习领域 / 一个孩子的成长 / 一段关系 / 一次旅程的 Memory 内快速记一段思考；（b）给已有录音 Segment 补一段笔记上下文（笔记是 audio Segment 的 Supplement）；（c）给已有笔记 Segment 补一段录音（录音是 note Segment 的 Supplement）。

**4. 用户想完成什么任务**
创建 note Segment 或 note Supplement → 在沉浸式编辑器内写 / 编辑 markdown body（含粘贴本地图片到 attachments/）→ 保存退出 → Memory Studio 自动可见 → 可重命名 / 删除 / 恢复。

**5. 页面或流程如何引导用户完成任务**
（a）Loaded Workspace 底部 ExpressionDock FAB 笔记 action 点亮（当前 disabled 升级为 enabled）；（b）Memory Studio selected Segment 右上 `+` 菜单提供"笔记补充"action；（c）Memory Studio 内容区 MarkdownContentSurface 右上 expand icon 进入沉浸式编辑已有笔记。沉浸式编辑器覆盖整个主区，不开新窗口；ESC dirty 时二次确认。保存路径单一：完成按钮或 Cmd+S。

**6. 需要突出的信息**
（a）FAB 笔记 action 视觉权重与录音 action 同；（b）沉浸式编辑器主区充满，让用户专注写作；（c）MarkdownContentSurface 默认只读预览，编辑入口用 expand icon 弱化但可见；（d）外部冲突时 banner + AlertDialog 主动暴露选择。

**7. 需要弱化的信息**
（a）audio kind 选中时 audio player 是主操作；note kind 选中时 player slot 显示弱化的占位文案"未来：笔记可被朗读"，不喧宾夺主；（b）笔记 More 菜单中"通过 agent 生成 prompt（暂不可用）"显示但 disabled，避免引导但承认未来方向；（c）笔记 Segment 不显示 duration（与 audio 区分）。

**8. 主操作**
（a）FAB 笔记 / `+` 笔记补充 / expand 进沉浸式；（b）沉浸式"完成"或 Cmd+S。

**9. 辅助操作**
（a）ESC 退出（dirty 二次确认）；（b）笔记 More 菜单：用默认应用打开 / 在访达中显示 / 复制相对路径 / 复制绝对路径 / 重命名 / 删除；（c）`+` 菜单内的"录音补充"与"笔记补充"互斥但平级。

**10. 风险操作**
（a）note Segment delete：复用 grace period undo toast（10s），自动关闭后才真实 trash；（b）note Supplement delete：真实 mutation + restore toast；（c）外部冲突"保留我的修改"：强制覆盖磁盘，丢失外部 agent 改动；（d）外部冲突"使用磁盘版本"：放弃 unsaved local body。

**11. 需要确认 / 提示 / 保护的地方**
（a）沉浸式 dirty ESC → AlertDialog 二次确认；（b）删除 Segment / Supplement → AlertDialog 二次确认 + 10s undo toast；（c）外部冲突 → banner + AlertDialog 两选项 + 取消；（d）draft 周期自动保存到 `.reo/drafts/segments/<id>/segment.md` + localStorage recovery marker；（e）应用启动检测 marker → 自动 mount NoteEditorOverlay；（f）attachment 粘贴 mime / size 校验失败 → toast + 移除 placeholder。

**12. 成功后用户会看到什么**
沉浸式编辑器关闭；Memory Studio 自动选中新 note Segment（create flow）或自动切到新 supplement tab（supplement create flow）；MarkdownContentSurface 立即渲染新 body；重启后 / 切回 Memory 后内容仍可见；外部 agent / Finder 打开 segment.md 可见 markdown + frontmatter。

**13. 失败后用户会看到什么**
（a）draft 周期保存失败：静默重试 3 次，最终失败 root toast；草稿仍在内存与磁盘（per N-1 retries）；（b）finalize / write 失败：toast 具体错误；overlay 回 normal/raw mode；草稿 + marker 保留；（c）外部冲突：banner + AlertDialog 而非 toast；（d）attachment 失败：toast + image block placeholder 移除；（e）lock lost：overlay 关闭 + toast "记忆空间已失效"；（f）recovery 失败：root toast "上次笔记恢复失败"，丢 marker。

**14. 功能边界**
范围内：multi-kind contract（schema + manifest + projection + Memory summary 重命名）、note Segment / Supplement create+edit+delete+restore+rename、Markdown Content Surface、NoteEditorOverlay 沉浸式编辑器、attachments + `reo-attachment://` protocol、external edit baselineContentHash 冲突处理。
范围外：自动 GC orphan attachments、file watcher、note kind 的 ASR / TTS / agent prompt 自动生成、video / photo / imported_file / html kind、widget runtime、Gallery、Prompt-bridge UI、AGENTS.md / users.md 模板、多窗口同步、公开发布 / auth / Sentry / updater / payment。

## 八、工程实现说明

**1. 功能如何进入**
（a）`ExpressionDock.tsx:36-40` FAB 笔记 action `disabled` → `onSelect: () => onStartNote()`；触发 App.tsx 的 `startNote` handler 派发 `workspace:createNoteSegmentDraft`，成功后 mount `NoteEditorOverlay`（mode=create）；（b）Memory Studio selected Segment `+` 菜单"笔记补充"action → `workspace:createNoteSegmentSupplementDraft` + mount overlay；（c）MarkdownContentSurface expand icon → mount overlay（mode=edit），先读 segment-content cache 计算 `baselineContentHash`。

**2. 功能如何退出**
（a）成功保存 → overlay close + cache 合并；（b）dirty ESC → AlertDialog；选丢弃 → `discardNoteSegmentDraft`（create flow only）/ revert（edit flow）+ 清 marker + close；选取消 → 回编辑器；（c）clean ESC → 直接 close；（d）lock lost / handle mismatch → 关闭 + 草稿保留；（e）App 关闭 / 崩溃 → marker 保留 → 下次启动 recovery。

**3. 状态如何流转**
见 §九 状态机。核心：loading → normal/raw → submitting → 成功 / conflict / 失败 → 回 normal/raw 或冲突解决。

**4. UI 元素展示规则**

- Segment.type='note' → Segment card 显示 line icon + 首段文本；Memory Studio player slot 显示占位文案；MarkdownContentSurface 渲染 note body
- Segment.type='audio' → 所有 audio 现状保留
- NoteEditorOverlay open → overlay layer 顶层，主区 frozen
- 笔记 More 菜单 → 转录组替换为"通过 agent 生成 prompt（disabled）"
- `+` 菜单 → 新增"笔记补充"item

**5. UI 元素隐藏规则**

- audio kind 选中时不显示 note placeholder
- note kind 选中时不挂 audio player
- recovery 时 overlay 自动 mount，不弹中间过渡 UI

**6. 控件可用 / 不可用规则**

- FAB 笔记：录音 overlay open / NoteEditorOverlay open 时 disabled
- 沉浸式"完成"按钮：submitting / 空 body 时 disabled
- 删除菜单项：mutation pending 时 disabled
- Conflict AlertDialog 两选项：处理中 disabled
- 切 Memory / sidebar / settings：NoteEditorOverlay open 时拦截 + root toast

**7. 控件 loading / disabled**

- FAB 笔记 IPC 在途时整组 disabled + 个体 spinner
- 沉浸式"完成" submitting → disabled + spinner
- MarkdownContentSurface outcome=loading → skeleton
- BlockNote 内 image block 上传中 → spinner placeholder

**8-9. 数据读取 / 写入**

- 读：`workspace:readSegmentContent`（generic，返回 discriminated by type；audio kind 含 audio bytes + transcript text；note kind 含 body text + byteLength + contentHash）、`workspace:readSegmentSupplementContent`、`workspace:readNoteSegmentDraftBody`（recovery 读）
- 写：`workspace:createNoteSegmentDraft` / `writeNoteSegmentDraftBody` / `finalizeNoteSegmentDraft` / `discardNoteSegmentDraft` + supplement 4 个同构 IPC
- Generic 写：`workspace:writeSegmentContent`（按 manifest kind 分派：audio 走 `replaceSegmentTranscript` + `markSegmentTranscriptionAttemptSuccess`；note 走 `replaceSegmentBody`；带 `baselineContentHash`）+ supplement 同构
- Attachments：`workspace:saveSegmentAttachment` / `listSegmentAttachments`
- `workspace:readFinalizedAudioSegment` / `...Supplement` 保留，仅服务 audio bytes 播放路径（transcript text 迁移到 generic channel）

**10. 数据同步**

- 仅 visibility refresh 重读：进入 visible → `workspace:readWorkspaceSnapshot` → Memory detail 后台刷新；dirty 时跳过对应 segment body 字段更新
- 不引入 file watcher
- 不引入云同步 / 多窗口同步

**11-12. 接口与异步结果**
17 个新 IPC channels（见 §13.1 接口表 / 设计 spec §6 §7 §13）。所有 mutation fire-and-forget；UI 不暴露取消；UI 30s 后允许用户关闭 overlay 但 mutation 继续。异步结果 settle 时：(a) 旧 workspaceHandle 提交的 response 由 renderer 静默丢弃（不改写新 session）；(b) success → cache merge；(c) failure → toast 或 AlertDialog；(d) conflict → AlertDialog；(e) draft 周期写失败 → 静默重试 3 次后 toast。

**13. 异常兜底**
见 §十二 边界情况补齐表。

**14. 权限判断**
当前阶段：trusted sender + workspaceHandle ownership + workspaceId 一致性 + lock usability，与现有 audio IPC 同标准。未来 Better Auth：role-based 占位，本 initiative 不实现。

**15. 字段建议**
（详见设计 spec §3 / §4 / §13）

- segment / supplement manifest 新增：`kind: 'audio' | 'note'`、`bodyByteLength: number`
- Memory summary 字段重命名：`durationMs` → `audioDurationMs`；`hasTranscript` → `hasAudioTranscript`；新增 `audioSegmentCount` / `noteSegmentCount` / `hasAnyNote`
- IPC 新字段：`body: string`、`baselineContentHash: string`、`revision: number`、`originalFilename: string`、`mimeType: enum`、`relativePath: string`
- 错误码新增：`ERR_SEGMENT_CONTENT_STALE`、`ERR_ATTACHMENT_UNSUPPORTED_MIME`、`ERR_ATTACHMENT_TOO_LARGE`、`ERR_ATTACHMENT_WRITE_FAILED`
- localStorage recovery marker 字段：`workspaceId / segmentId | supplementId+parentSegmentId+targetKind / title / bodySnapshot / baselineContentHash / dirty / updatedAt`

**16. 性能注意**

- BlockNote mount 较大 markdown (~50KB) 不卡顿（spike #1 实测）
- baselineContentHash sha256 单次 ~5KB markdown < 1ms（spike #5 实测）
- attachments 25 MiB IPC payload 通过 Electron 主进程 streaming（如果 Electron IPC 单次 payload 限制不允许，spike #3 验证后调降到 5 MiB 或改 chunked upload）
- draft throttled write 400ms idle；防止 typing 高频触发
- Segment strip windowing 现有规则不动，跨 kind item 不增加 mount 数

**17. 防丢失**
draft 写入 `.reo/drafts/segments/<id>/segment.md` 或 `.reo/drafts/supplements/<id>/supplement.md` + localStorage marker 同步；崩溃 / 强退 / 重启可恢复；ESC dirty AlertDialog；conflict 选择前本地 body 不被覆盖。

**18. 防重复提交**
mutation pending 时 button disabled；FAB 整组 disabled；Cmd+S 触发器 pending 时忽略；冲突 AlertDialog 选项处理中 disabled。

**19. 防旧结果串写**
所有 mutation 提交时捕获 `workspaceHandle`；in-flight response 若 handle 不匹配当前 session → renderer 静默丢弃，不改写 cache / projection / session；helper `segmentContentQueryBelongsToWorkspace`（重命名自 `finalizedAudioContentQueryBelongsToWorkspace`）按 workspaceId 清同 workspace 的 content cache；进入新 session 时清同 workspaceId 的内容 cache（与现有 audio 路径一致）。

**20. 验收标准**
见 §十三 验收。`npm run verify:quick` 必须全绿；6 个 spike 必须在 sub-spec (a) 代码改动前完成；每个 sub-spec phase 必须通过 `/review` + `$ycksimplify` 双 gate。
