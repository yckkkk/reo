# 商业化基础设施里程碑

## 当前状态

商业化基础设施 initiative 已创建，当前没有对应实现 spec。求职阶段主攻 AI 产品经理，增长产品经理作为副线。

`docs/specs/` 已有进行中的工作单元时，不创建商业化 spec，除非用户明确要求并行规划 spec。商业化 initiative 可以保持 active，等待当前工作单元收口后继续拆分。

当前执行准则：先做可演示 AI 工作流、AI eval、内测反馈和 case study；商业化基础设施只保留为 public paid release gate，直到需求、内测和分发信号成立。AI-native、FDE-shaped、human-AI collaboration、agent-ready memory space、Codex-style 创作和 model release review 只能服务 demo、eval、内测包和 case study，不再扩成新的主线。

## 里程碑

- [ ] C0：完成 AI 产品经理求职作品集定位，明确 Reo 的目标用户、核心场景、不做清单和面试叙事。
- [ ] C1：完成可演示 AI 工作流，覆盖输入材料、结构化洞察、证据引用和失败恢复。
- [ ] C2：完成 AI eval 与成本评估，覆盖样本集、质量维度、幻觉率、延迟和成本。
- [ ] C3：完成产品 case study，覆盖用户、问题、竞品、取舍、MVP、指标、风险和路线图。
- [ ] C4：完成内测运营与增长副线材料，覆盖内测群、招募问卷、任务脚本、反馈闭环、landing page 或 waitlist、激活漏斗和实验 backlog。
- [ ] C5：将 AI-native PM 与 FDE-shaped 能力压缩进主交付物或 appendix，覆盖 operating model、human-AI collaboration map、field loop、model release review。
- [ ] C6：完成 Reo 产品创新主张，覆盖 Record to Create loop、agent-ready memory space、Codex-style 创作、初始化 `AGENTS.md`/skills/plugins、设计系统约束 artifact。
- [ ] C7：完成面试包，覆盖演示视频、简历 bullet、作品集页面和面试讲稿。
- [ ] C8：完成 2/4/8 周求职执行路线的阶段复盘，每周只验收 demo 稳定性、eval 样本增长和真实用户任务反馈。
- [ ] M0：完成开源复用评估和商业化架构选择。
- [ ] M1：确定公开付费发布的合规、备案、支付、短信、微信开放平台和域名门禁。
- [ ] M2：确定身份服务方案，覆盖 Electron auth、手机号、微信、设备、session 和账号合并。
- [ ] M3：确定会员、entitlement、credit ledger 和套餐模型。
- [ ] M4：确定 AI Gateway、豆包语音 ASR server-side adapter 和未来 LLM Gateway 边界。
- [ ] M5：确定火山引擎消费治理、项目/标签、预算、账单、调用量和对账机制。
- [ ] M6：确定支付、订单、退款、发票、加量包和后台运营边界。
- [ ] M7：确定 SEO、公开站点、下载页、帮助中心、遥测和隐私策略。
- [ ] M8：完成 public paid release gate，连接 packaging、signing、updater、download 和运营监控。
