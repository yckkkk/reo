# Reo 产品定位与核心叙事

时间：2026-05-18 America/Los_Angeles

本文档承担 Reo 的外向定位叙事职责：核心 thesis、用户场景、电梯演讲、唯一性论证。技术决策见 `docs/decisions/0006-agent-native-carrier-and-generative-ui.md`，实现真源见 `docs/current/product.md`，商业化叙事见 `product-thesis.md`，竞品地图见 `competitive-analysis.md`。

## 核心 Thesis

> **Reo 是让你围绕一件事产生的所有材料，被 agent 转化成只属于你的作品的私人空间。**

四个关键词同时成立才是 Reo：

1. **"围绕一件事"** — 主题容器（Memory），不是时间流，不是文件夹，不是项目管理
2. **"所有材料"** — 多模态：录音、视频、照片、笔记、上传文件平等存在
3. **"agent 转化"** — 不是存储 / 检索 / 搜索，是 agent 生成、重构、创作
4. **"只属于你的作品"** — 唯一性。别人无法复制（材料是你的），你自己做不到（没有 agent 替你做）

任何一条单独都不构成 Reo。Apple Photos 有第二条，缺其他三条。Voice Memos 有第三条的影子，缺其他。Notion 有第一条的近似，缺其他。Day One 有前两条，缺第三和第四。Cursor 有第三条，但用户和场景偏开发者。

## 用户场景

Reo 的价值在跨模态主题容器中长期成立。具体形态：

  视频/

## 用户的"使用 Reo 一年后"

不是一堆录音。是这样的东西：

- 5-10 个 Memory，每个都是你长期关心的主题
- 每个 Memory 里有不同模态的材料持续累积
- 每个 Memory 已经被 agent 转化成 3-5 个可看的 widget 和可读的 HTML 作品
- Gallery 走马灯让你定期遇到过去
- 所有内容在你本地，可以备份、可以迁移、不会因为某个公司倒闭消失

这是 Reo 的产品形态终局。

## Reo 替代什么 / 启用什么

### Reo 替代的

| 你现在用的 | Reo 替代的角色 |
|---|---|
| Voice Memos | 录音夹（但 Reo 给录音一个主题家） |
| Apple Photos / 相册 | 媒体堆（但 Reo 按主题聚合） |
| Notion 草稿 | 半成品笔记（但 Reo 把它变成 agent 可操作的素材） |
| 桌面文件夹 | 参考资料堆（但 Reo 让它和你的思考在同一个容器里） |
| 微信收藏 | 灵感收集器（但 Reo 不依赖云端） |
| 半完成的博客草稿 | 想表达没完成的内容（但 Reo 让 agent 帮你完成） |

### Reo 启用的（其他工具做不到的）

- **跨模态在主题里相遇**：你的录音和你的照片和你的笔记在同一个语义容器里
- **agent 在你完整的主题上工作**：不是单条笔记的总结，是整个主题的转化
- **唯一作品**：别人做不出来（材料是你的），你做不出来（没 agent 替你做）
- **不损失自己**：所有材料是本地文件，平台倒闭不影响你
- **回到过去**：Gallery 走马灯让你和你的累积重逢

## 唯一性 Moat

Reo 的护城河不是某个功能，是 **agent 在你长期累积的多模态主题材料上生成的作品具有不可复制性**：

- **其他工具 + AI** 也能生成内容，但用的是公共数据，输出不是你的，是同质的
- **同一个 agent + 别人的材料** 输出不一样，因为材料决定了输出
- **你的材料 + 没有 agent** 永远停在材料阶段，不会变成作品
- **你的材料 + 你自己** 没有时间做整理和创作

Reo 让 "你的材料 + agent" 这个组合成为可能。这个组合的输出是真正属于你的作品。

这层护城河随时间加深：你在 Reo 累积越多，可生成的作品维度越多，迁移到任何其他工具都不可能复制这层价值。这是真正的 lock-in，但是是基于用户拥有的 lock-in，不是基于平台扣留的 lock-in。

## 电梯演讲

### 10 秒

> Reo 让你围绕一件事产生的所有材料——录音、视频、照片、笔记、上传——成为 agent 能转化成你的作品的本地素材。

### 30 秒

> 你对一件事产生的素材永远是分散的：录音在 Voice Memos，照片在 Photos，参考资料在文件夹，笔记在 Notion。它们永远没有变成什么。
>
> Reo 是按主题积累所有形式材料的本地空间。你的录音、视频、照片、笔记、上传文件全部进入同一个 Memory 容器。Codex 或 Claude Code 这种 agent 在你整个容器上工作，生成 widget、HTML 作品、走马灯回顾——这些东西别人做不出来，因为材料是你的。
>
> 别的工具帮你存，Reo 帮你把存的东西变成你的作品。

### 60 秒

> 你对一件事有持续的关注——你在写的一本书、你的孩子的成长、你在学的一个领域、你和某个人的关系——你会产生大量素材：录音、视频、照片、笔记、参考文件。但这些素材永远是分散的，存在六个不同的 app 里，永远没有变成什么。
>
> Reo 是按主题积累所有形式材料的本地空间。每个你长期关心的主题开一个 Memory，所有相关的录音、视频、照片、笔记、上传文件都在同一个本地文件夹里成长。
>
> 真正不一样的是：Codex 或 Claude Code 在你整个 Memory 的全部材料上工作。给你的孩子成长 Memory 生成成长册 HTML，给你的书生成章节大纲 widget，给你的学习 Memory 生成概念时间线，给你和某人的对话生成关系曲线 widget。Gallery 走马灯让你的过去定期回到你眼前。
>
> 这些作品别的工具做不出来，因为它们用的是你的材料。你自己做不出来，因为没有 agent 替你做这些重活。Reo 让两件事相遇。
>
> 别的工具帮你"存"，Reo 帮你"做"。一切都在你本地，一切都属于你。

## Founder voice 起点

第一篇创作者公开输出可以从这里开始：

> 我看到 Flomo 把 6 张 AI 视角卡片做成付费功能（默认洞察、价值澄清、二阶思考、逆向思考、CBT、MBTI）。意识到 Reo 应该做的不是更多 AI 视角，而是更深的事：让你产生的所有材料——录音、视频、照片、笔记、上传——按主题积累，让 Codex 这种 agent 在这些材料上替你生成只属于你的作品。Reo 是这样一个产品。

或者从用户场景开始：

> 我的孩子两岁了。两年里我产生了三百段录音、几千张照片、几十段视频、上百条笔记，分散在六个 app 里。我做不出一本属于她的成长册——不是因为我懒，是因为没有任何工具能在所有材料上工作。Reo 是为了让 agent 能在我所有材料上工作而做的。
